## Callout

<Callout>
  TinaCMS helps editors manage content, while Fumadocs turns it into beautiful documentation.
</Callout>

<Callout type="warn">
  Make sure your MDX components are registered before using them in docs pages.
</Callout>

<Callout type="error">
  Do not edit generated content files directly if TinaCMS is managing them.
</Callout>

## Accordion

<Accordions type="single">
  <Accordion title="Why use TinaCMS with Fumadocs?">
    TinaCMS gives editors a visual editing experience, and Fumadocs gives developers a clean docs framework.
  </Accordion>

  <Accordion title="Can I still write Markdown?">
    Yes. You can write normal Markdown and add Fumadocs UI components when needed.
  </Accordion>

  <Accordion title="Who is this good for?">
    It works well for product docs, developer docs, internal guides, and content-heavy websites.
  </Accordion>
</Accordions>
